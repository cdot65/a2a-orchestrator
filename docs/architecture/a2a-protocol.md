# A2A Protocol

This project targets the [Agent-to-Agent (A2A)](https://github.com/google/agent-to-agent) protocol via the official `a2a` Python SDK. Agents are reachable over plain HTTP and speak streaming JSON-RPC.

## Surface per agent

Every agent exposes:

| Path | Purpose |
|---|---|
| `GET /.well-known/agent-card.json` | Discovery — returns the agent's `AgentCard` |
| `POST /` | JSON-RPC entrypoint for `message/send` and `message/stream` |

The orchestrator additionally exposes:

| Path | Purpose |
|---|---|
| `GET /v1/models` | OpenAI-compat model listing |
| `POST /v1/chat/completions` | OpenAI-compat chat endpoint |
| `GET /v1/openapi.json` | OpenAI-compat surface only (FastAPI auto-generated) |
| `GET /v1/docs` | Swagger UI for OpenAI-compat surface |
| `GET /openapi.json` | Full merged spec (A2A + OpenAI-compat) |

## Message flow

A streaming request from a client:

```json
{
  "jsonrpc": "2.0",
  "id": "1",
  "method": "message/stream",
  "params": {
    "message": {
      "role": "user",
      "parts": [{"kind": "text", "text": "Give me a vegan ramen recipe."}],
      "messageId": "m1"
    }
  }
}
```

The server responds with a stream of JSON-RPC envelopes, each wrapping one of:

| Event type | Meaning |
|---|---|
| `TaskStatusUpdateEvent` (state=`working`, with `message`) | Interim text update — orchestrator forwards every child-agent event this way |
| `TaskStatusUpdateEvent` (state=`working`, no message) | Stage marker (e.g., `discovering`, `planning`, `synthesizing`) |
| `TaskArtifactUpdateEvent` | Final structured output (a recipe JSON, a shell-output JSON) |
| `TaskStatusUpdateEvent` (state=`completed`, `final=true`) | Terminal frame |
| `TaskStatusUpdateEvent` (state=`failed`/`canceled`, `final=true`) | Error or cancel terminal |

The orchestrator's `dispatch_step()` (see `orchestrator/executor.py`) consumes the child-agent stream, treats `failed` as `RuntimeError`, treats `canceled`/`rejected`/`input-required`/`auth-required` as terminal-non-completed errors, and returns the **last** artifact text seen.

## Sending child-agent calls

The orchestrator uses `a2a.client.A2AClient` to talk to its peers:

```python
from a2a.client import A2AClient
from a2a.types import (
    Message, MessageSendParams, Role,
    SendStreamingMessageRequest, TextPart,
)

async with httpx.AsyncClient(timeout=None) as http:
    client = A2AClient(httpx_client=http, url=agent_url)
    request = SendStreamingMessageRequest(
        id=os.urandom(8).hex(),
        params=MessageSendParams(
            message=Message(
                role=Role.user,
                parts=[TextPart(text=input_text)],
                message_id=os.urandom(8).hex(),
            )
        ),
    )
    async for wrapper in client.send_message_streaming(request):
        ...
```

`wrapper.root` is either a `JSONRPCErrorResponse` or a successful response whose `result` is one of the event types above.
